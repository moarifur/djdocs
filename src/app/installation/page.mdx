# 🎓 LMS Platform

A modern, full-featured Learning Management System built with cutting-edge web technologies. Create, manage, and deliver online courses with integrated video processing, secure payments, and comprehensive progress tracking.

## ✨ Features

### 🎯 Core Functionality
- **Course Discovery** - Browse and filter courses with advanced search
- **Secure Payments** - Integrated Stripe payment processing
- **Progress Tracking** - Mark chapters complete/incomplete with visual progress indicators
- **Student Dashboard** - Personalized learning experience with course overview
- **Instructor Tools** - Complete course creation and management suite

### 🛠️ Advanced Features
- **Drag & Drop** - Intuitive chapter reordering for course creators
- **File Uploads** - Seamless file management with UploadThing integration
- **Video Processing** - Professional video handling with Mux
- **HLS Video Player** - Adaptive streaming for optimal viewing experience
- **Rich Text Editor** - WYSIWYG content creation
- **Authentication** - Secure user management with Clerk
- **Route Protection** - Role-based access control

## 🏗️ Architecture & Component Structure

### Route Organization
The project uses Next.js 13+ App Router with advanced routing patterns:

```
app/
├── (auth)/                    # Route group - no URL segment
│   └── (routes)/             # Nested route group
│       └── sign-in/          # /sign-in
├── (dashboard)/              # Dashboard route group
│   ├── (routes)/            
│   │   ├── page.mdx         # /dashboard home
│   │   └── search/          # /search
│   ├── _components/         # Private folder (underscore prefix)
│   │   ├── sidebar.jsx      # Main sidebar component
│   │   ├── sidebar-routes.jsx # Dynamic route rendering
│   │   ├── sidebar-item.jsx  # Individual nav items
│   │   ├── nav.jsx       # Top navigation
│   │   ├── mobile-sidebar.jsx # Responsive hamburger menu
│   │   └── logo.jsx         # Brand logo component
│   └── layout.jsx           # Dashboard wrapper layout
├── teacher/                  # Teacher interface
│   ├── courses/             # /teacher/courses
│   └── analytics/           # /teacher/analytics
└── components/              # Shared components
    └── navbar-routes.jsx    # Context-aware navigation
```

### Component Hierarchy & Relationships

#### Authentication Flow
```
Root Layout (app/layout.jsx)
├── ClerkProvider (wraps entire app)
├── Header (sign-in/sign-up buttons)
└── Page Content
```

#### Dashboard Layout System
```
Dashboard Layout (app/(dashboard)/layout.jsx)
├── Nav Component
│   ├── MobileSidebar (hamburger menu)
│   └── NavbarRoutes (context-aware buttons)
├── Sidebar Component (desktop)
│   ├── Logo Component
│   └── SidebarRoutes
│       └── SidebarItem[] (mapped navigation items)
└── Main Content Area
```

### Key Component Features

#### Dynamic Navigation System
- **Context-Aware Routes**: Different navigation items for students vs teachers
- **Active State Management**: Visual indicators for current page
- **Responsive Design**: Desktop sidebar + mobile hamburger menu
- **Role-Based Access**: Automatic switching between guest and teacher modes

#### Authentication Integration
- **Global Auth Context**: Clerk provider at root level
- **Route Protection**: Middleware-based authentication
- **Conditional UI**: Sign-in/Sign-up vs User profile button
- **Secure Redirects**: Post-authentication routing

## 🔧 Installation

### Prerequisites
- Node.js 18+
- npm or yarn package manager
- Git

### Quick Start

1. **Create Next.js project with Tailwind CSS**
```bash
npx create-next-app@latest lms-platform --tailwind --eslint --app
cd lms-platform
   ```

2. **Initialize Shadcn UI**
```bash
npx shadcn@latest init
   ```

3. **Install required Shadcn components**
```bash
# Sheet component for mobile sidebar
npx shadcn@latest add sheet

# Button component for navigation
npx shadcn@latest add button
   ```

4. **Install Clerk Authentication**
```bash
npm install @clerk/nextjs
   ```

5. **Set up environment variables**
Create `.env.local` in your root directory:
```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_your_publishable_key_here
CLERK_SECRET_KEY=sk_test_your_secret_key_here
   ```

6. **Add global CSS styling**
Add to your `app/globals.css` after Tailwind imports:
```css
@tailwind base;
@tailwind components;
@tailwind utilities;

/* Ensure full height layout */
html, body, :root {
  height: 100%;
}
   ```

7. **Start development server**
```bash
npm run dev
   ```

8. **Open in browser**
Navigate to [http://localhost:3000](http://localhost:3000)

```javascript
// middleware.js

export default clerkMiddleware();

    '/(api|trpc)(.*)',
  ],
};
```

## 🐛 Troubleshooting

### Common Issues

#### npm EEXIST Error
If you encounter npm installation conflicts, try these solutions:

| Method | Steps | When to Use | Pros | Cons |
|--------|-------|-------------|------|------|
| **Force Overwrite** | `npm install -g npm@latest --force` | Quick fix needed | Fastest solution | May cause rare issues |
| **Manual Deletion** | Delete `%AppData%\npm\npx.cmd` then run `npm install -g npm@latest` | If --force fails | Clean update | Manual intervention |
| **Clear Cache** | `npm cache clean --force` then retry | Cache corruption | Fixes cache issues | May not resolve conflicts |
| **Windows Upgrade Tool** | `npm install -g npm-windows-upgrade` then `npm-windows-upgrade --version latest` | Windows-specific | Reliable for Windows | Extra tool needed |

#### Styling Issues
Ensure full height layout by adding to `globals.css`:


### Development Guidelines

#### Code Organization Principles
- **Route Groups**: Use `(auth)`, `(dashboard)` for logical organization without URL impact
- **Private Folders**: Use `_components` (underscore prefix) for component organization
- **Colocation**: Keep related components close to their usage context
- **Separation of Concerns**: Clear distinction between layouts, pages, and components

#### Component Architecture Patterns
```javascript
// ✅ Good: Client component for interactivity
"use client"
import { usePathname } from "next/navigation";

// ✅ Good: Server component for static content (default)
export default function Layout({ children }) {
  return <div>{children}</div>
}

// ✅ Good: Clear prop destructuring
const SidebarItem = ({ icon: Icon, label, href }) => {
  // Component logic
}
```

#### Styling Guidelines
- **Tailwind Utility Classes**: Consistent spacing (`p-4`, `gap-x-2`, `h-full`)
- **Conditional Styling**: Use `cn()` utility from `lib/utils` for dynamic classes
- **Responsive Design**: Mobile-first with `md:` breakpoints
- **Color Consistency**: Sky theme for active states, slate for neutral

```javascript
// ✅ Good: Conditional styling with cn()
className={cn(
  "base-styles",
  isActive && "active-styles"
)}
```

#### State Management Patterns
- **URL-Based State**: Use `usePathname()` for navigation context
- **Component State**: Local state for UI interactions
- **Global State**: Clerk provider for authentication context
- **Route Context**: Dynamic routing based on pathname patterns

#### Next.js Best Practices
- **App Router**: Leverage layouts and route grouping
- **Image Optimization**: Use `next/image` for all images
- **Navigation**: Use `useRouter()` for programmatic navigation
- **Middleware**: Global route protection and authentication
