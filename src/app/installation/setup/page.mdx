## тЬЕ Step 2: Initialize a Git Repository

Let's now initialize version control with Git.

Navigate to your project folder:

```bash
cd lms-client
```

Initialize Git and make your first commit:

```bash
git init
git add .
git commit -m "Development Base has been Created"
```

Next, connect your local project to a GitHub repository:

```bash
git remote add origin https://github.com/moarifur/lumilearn-client.git
git branch -M main
git push -u origin main
```

ЁЯОЙ Now your base project is live on GitHub!

## тЬЕ Step 3: Set Up Git Branches

We follow the Git Feature Branch Workflow. Let's set up the dev branch and the first feature branch:

```bash
git checkout -b dev
git checkout -b feature/EPIC-1-project-initialization-auth
```

Verify your branches:

```bash
git branch
```

Output:

```
  dev
* feature/EPIC-1-project-initialization-auth
  main
```

Perfect! You're now ready for isolated development.

### ЁЯФД Git Feature Workflow

Branch: feature/EPIC-1-project-initialization-auth

When we're in the feature branch, we're gonna make changes and doing stuff and then we have to commit it before go to another branch, otherwise we can't merging anything at all.

##### So, for that we need to do this

```bash
git add .
git commit -m "Add new changes"
```

After commiting to that branch (e.g.: feature/EPIC-1-project-initialization-auth) then we need to merge it to the dev branch
#### So, Merging to Dev:

```bash
git checkout dev
git pull origin dev
git merge feature/EPIC-1-project-initialization-auth
git push origin dev
```

##### And this is how we are gonna handle the upstream branch with downstream branch
