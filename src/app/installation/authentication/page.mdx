# 🔐 Authentication with Clerk in Next.js 14 (App Router)

Implementing authentication in a modern app shouldn't be hard. With Clerk, it's straightforward, secure, and beautifully customizable. This guide walks you through the entire setup process, from registering your app to creating custom sign-in and sign-up pages.

## 🧭 Step 1: Create Your Clerk Application

1. Go to: [Clerk.com](https://clerk.com)
2. Sign in / Sign up
3. Navigate to the Clerk Dashboard
4. Click "Create Application"
5. **Application Name:** lumilearn
6. **Sign-in options:** ✅ Google ✅ GitHub ✅ LinkedIn
7. Click "Create application"
8. You'll be redirected to your app instance

## ⚙️ Step 2: Set Up Clerk in Your Next.js App

### 1. Install Clerk SDK

```bash
npm install @clerk/nextjs
```

### 2. Add Environment Variables

Create a `.env` file at the root of your project and add:

```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=your_publishable_key
CLERK_SECRET_KEY=your_secret_key
```

🔑 You can find these in the API Keys section of your Clerk dashboard.

## 🧱 Step 3: Configure middleware.js

At the root or inside `/src`, create `middleware.js`:

```javascript
import { clerkMiddleware } from '@clerk/nextjs/server';

export default clerkMiddleware();

export const config = {
  matcher: [
    '/((?!_next|[^?]*\\.(?:html?|css|js(?!on)|jpe?g|webp|png|gif|svg|ttf|woff2?|ico|csv|docx?|xlsx?|zip|webmanifest)).*)',
    '/(api|trpc)(.*)',
  ],
};
```

## 🧩 Step 4: Add ClerkProvider in Root Layout

Update your `src/app/layout.js` to wrap the app with ClerkProvider:

```javascript
import {
  ClerkProvider,
  SignInButton,
  SignUpButton,
  SignedIn,
  SignedOut,
  UserButton,
} from '@clerk/nextjs'

export default function RootLayout({ children }) {
  return (
    <ClerkProvider>
      <html lang="en">
        <body>
          <header className="flex justify-end p-4 gap-4 h-16">
            <SignedOut>
              <SignInButton />
              <SignUpButton>
                <button className="bg-[#6c47ff] text-white rounded-full px-5 py-2">
                  Sign Up
                </button>
              </SignUpButton>
            </SignedOut>
            <SignedIn>
              <UserButton />
            </SignedIn>
          </header>
          {children}
        </body>
      </html>
    </ClerkProvider>
  )
}
```

## 👤 Step 5: Run the Project & Create First User

```bash
npm run dev
```

Visit `http://localhost:3000` and sign up using Clerk's pre-built UI.

## ✍️ Step 6: Build Custom Sign-In and Sign-Up Pages

### 📄 Create File Structure

```
src/
└── app/
    └── (auth)/
        ├── sign-in/
        │   └── [[...sign-in]]/
        │       └── page.js
        └── sign-up/
            └── [[...sign-up]]/
                └── page.js
```

### ✅ sign-in/page.js

```javascript
import { SignIn } from '@clerk/nextjs'

export default function Page() {
  return <SignIn />
}
```

### ✅ sign-up/page.js

```javascript
import { SignUp } from '@clerk/nextjs'

export default function Page() {
  return <SignUp />
}
```

## 💅 Step 7: Center the Auth UI with a Layout

Create `layout.js` in the `(auth)` folder:

```javascript
const AuthLayout = ({ children }) => {
  return (
    <div className="h-screen flex items-center justify-center">
      {children}
    </div>
  );
};

export default AuthLayout;
```

## 🔓 Step 8: Make Auth Routes Public

Update `middleware.js`:

```javascript
import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server'

const isPublicRoute = createRouteMatcher([
  '/sign-in(.*)',
  '/sign-up(.*)'
]);

export default clerkMiddleware(async (auth, req) => {
  if (!isPublicRoute(req)) {
    await auth.protect()
  }
})

export const config = {
  matcher: [
    '/((?!_next|[^?]*\\.(?:html?|css|js(?!on)|jpe?g|webp|png|gif|svg|ttf|woff2?|ico|csv|docx?|xlsx?|zip|webmanifest)).*)',
    '/(api|trpc)(.*)',
  ],
}
```

## 🌐 Step 9: Update More Env Variables

```env
# Sign In
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_IN_FALLBACK_REDIRECT_URL=/

# Sign Up
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
NEXT_PUBLIC_CLERK_SIGN_UP_FALLBACK_REDIRECT_URL=/
```

## ✅ Final Check

Run your app:

```bash
npm run dev
```

Visit:
- `http://localhost:3000/sign-in`
- `http://localhost:3000/sign-up`

Now you have fully working custom sign-in and sign-up pages with protected routes powered by Clerk + Next.js 14 App Router.
