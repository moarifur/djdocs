# Data Relationships

In real-world applications, data rarely exists in isolation. Entities are connected to one another, and understanding these connections is essential for designing a proper data model.

## What Is a Data Relationship?

A data relationship describes how two data entities are connected. For example, consider a system that tracks **books** and **authors** — two distinct entities that are clearly related: a book belongs to an author, and an author can write books.

These connections between entities are called **relationships**, and there are three primary types.

---

## Types of Relationships

### One-to-Many

A **one-to-many** relationship exists when one record in entity A can be associated with multiple records in entity B, but each record in entity B belongs to only one record in entity A.

**Example:** An author can write many books, but each book is written by one author.

> J.K. Rowling wrote multiple Harry Potter books as well as other unrelated titles — but each individual book has a single author.

This is the most common relationship type and appears in virtually every project.

---

### One-to-One

A **one-to-one** relationship exists when one record in entity A is associated with exactly one record in entity B, and vice versa.

**Example:** An author has one address, and that address belongs to that author.

This relationship type is less common but useful for separating concerns or isolating optional data.

---

### Many-to-Many

A **many-to-many** relationship exists when multiple records in entity A can relate to multiple records in entity B.

**Example:** A book can be published in many countries, and a country can publish many books.

| Perspective | Relationship |
|-------------|--------------|
| One book → countries | One to Many |
| One country → books | One to Many |
| Combined | **Many to Many** |

---

## Summary

| Relationship Type | Description | Example |
|-------------------|-------------|---------|
| One-to-One | Each record maps to exactly one other | Author ↔ Address |
| One-to-Many | One record maps to many others | Author → Books |
| Many-to-Many | Many records map to many others | Books ↔ Countries |

---

## Which Types Will You Use?

Not every project requires all three relationship types:

- **One-to-many** is present in almost every project.
- **Many-to-many** is also common and often appears alongside one-to-many.
- **One-to-one** is used occasionally, depending on your data model.

Understanding which relationships exist in your data is the first step toward connecting your models correctly.
