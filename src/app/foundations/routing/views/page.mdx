# Views

> **Definition:** A View is a function or class that handles a user request, runs business logic, and returns a response (HTML, JSON, redirect, or a file).

```
URL = where the user goes
View = what happens there
```

## Overview

A **view** is a function or class that receives a web request, executes business logic, and returns a response. Every URL in a Django project maps to exactly one view.

## What a view can do

A view is not limited to a single purpose. It can:

- Show plain text
- Render an HTML page
- Fetch data from the database
- Process form input
- Return JSON (for React or other front-ends)
- Return a redirect or a file download

## Minimal example

The simplest possible view returns a plain text response using `HttpResponse`.

```python filename="challenges/views.py"
from django.http import HttpResponse

def index(request):
    return HttpResponse('Hello!')
```

## Return types

Django views can return several different response types depending on the use case.

| Return type | Use case         | Example                                               |
| ----------- | ---------------- | ----------------------------------------------------- |
| HTML        | Web pages        | `return render(request, 'template.html', context)`    |
| JSON        | APIs / React     | `return JsonResponse({'key': 'value'})`               |
| Redirect    | Send user elsewhere | `return redirect('/other-page/')`                  |
| File        | Download a file  | `return FileResponse(open('file.pdf', 'rb'))`         |
| Plain text  | Quick testing    | `return HttpResponse('Hello!')`                       |
