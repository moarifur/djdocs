# Book Detail View

In this guide, you'll build a clickable book list that navigates to individual book detail pages. Each detail page will display the book's title, author, rating, and bestseller status.

---

## 1. Create the Detail Template

Create `book_outlet/templates/book_outlet/book_detail.html`:

```html
{% extends "book_outlet/base.html" %}

{% block title %}{{ title }}{% endblock %}

{% block content %}
  <h1>{{ title }}</h1>
  <h2>{{ author }}</h2>
  <p>
    The book has a rating of {{ rating }} and
    {% if is_bestseller %}
      is a best seller.
    {% else %}
      but isn't a best seller.
    {% endif %}
  </p>
{% endblock %}
```

> **Note:** The template uses three context variables — `title`, `author`, `rating`, and `is_bestseller` — all of which must be passed from the view.

---

## 2. Create the Detail View

In `book_outlet/views.py`, add the `book_detail` view:

```python
from django.shortcuts import render, get_object_or_404
from .models import Book

def book_detail(request, id):
    book = get_object_or_404(Book, pk=id)

    return render(request, "book_outlet/book_detail.html", {
        "title": book.title,
        "author": book.author,
        "rating": book.rating,
        "is_bestseller": book.is_bestselling,
    })
```

### How it works

- The view accepts an `id` parameter from the URL.
- `get_object_or_404(Book, pk=id)` fetches the book by primary key. If no book is found, it automatically returns a **404 response**.
- The fetched model fields are mapped to context keys that match what the template expects.

> **Tip:** Django models have an auto-incrementing integer `id` field set as the primary key by default. You can use `pk` as an alias for whichever field is the primary key — in this case, `id`.

---

## 3. Register the URL

In `book_outlet/urls.py`, add a dynamic path for the detail page:

```python
from django.urls import path
from . import views

urlpatterns = [
    path("", views.index, name="index"),
    path("<int:id>/", views.book_detail, name="book-detail"),
]
```

### Dynamic URL segments

- `<int:id>` captures a segment from the URL and converts it to an **integer** before passing it to the view.
- The name `id` must match the parameter name in your view function (`def book_detail(request, id)`).
- Using `int:` ensures the URL only matches numeric values, and automatically coerces the string to an integer.

## 4. Handle 404 Errors

### Option A — Manual try/except

```python
from django.http import Http404
from .models import Book

def book_detail(request, id):
    try:
        book = Book.objects.get(pk=id)
    except Book.DoesNotExist:
        raise Http404()

    return render(request, "book_outlet/book_detail.html", { ... })
```

### Option B — `get_object_or_404` (recommended)

```python
from django.shortcuts import render, get_object_or_404

def book_detail(request, id):
    book = get_object_or_404(Book, pk=id)
    ...
```

`get_object_or_404` is a Django shortcut that combines the try/except pattern into a single line. Import it from `django.shortcuts`.

> If you have a `404.html` template in your templates directory, Django will render it automatically when an `Http404` is raised.

## Common Mistakes

**Wrong field name on the model**

If your model field is `is_bestselling` but you access `book.is_bestseller` in the view, Django will raise an `AttributeError`. The **context key** in the view (e.g., `"is_bestseller"`) can be named anything — but the **model attribute** you read from must exactly match the field defined in `models.py`.

```python
# ✅ Correct — reads the right model field
"is_bestseller": book.is_bestselling,

# ❌ Wrong — AttributeError if field is named is_bestselling
"is_bestseller": book.is_bestseller,
```
