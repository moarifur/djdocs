# Slug Fields & Overriding the Save Method

## Overview

This guide covers two important Django model techniques:

- Adding a **slug field** to your model for cleaner, SEO-friendly URLs
- **Overriding the `save()` method** to auto-populate the slug from the title
- Running **migrations** after model changes
- Using Django's shell to update existing records

---

## Why Use Slugs?

Instead of URLs like `/books/1`, slugs give you human-readable URLs like `/books/harry-potter-one`. This is better for users and for search engines.

<Callout type="info">
    The slug concept is the same one used in standard blog applications — a URL-safe version of a title string.
</Callout>

---

## Step 1: Add a `SlugField` to Your Model

Django has a built-in `SlugField` that validates slug-formatted strings automatically.

```python filename="book_outlet/models.py"
from django.db import models
from django.utils.text import slugify


class Book(models.Model):
    title = models.CharField(max_length=200)
    # ... other fields ...

    slug = models.SlugField(unique=True)
```

**Key points:**
- `SlugField` enforces slug formatting (e.g. `harry-potter-one`)
- `unique=True` ensures the database enforces that no two records in that table have the same slug

---

## Step 2: Override the `save()` Method

To auto-populate the slug whenever a book is saved, override the model's `save()` method.

```python filename="book_outlet/models.py"
from django.db import models
from django.utils.text import slugify


class Book(models.Model):
    title = models.CharField(max_length=200)
    slug = models.SlugField(unique=True)

    def save(self, *args, **kwargs):
        self.slug = slugify(self.title)
        super().save(*args, **kwargs)
```

<Callout type="warning">
    Always call `super().save(*args, **kwargs)` inside your overridden method. Without it, Django's built-in save logic (which writes to the database) will never run.
</Callout>

**How it works:**

| Step | What happens |
|------|-------------|
| `save()` is called | Your custom method runs first |
| `slugify(self.title)` | Converts `"Harry Potter One"` → `"harry-potter-one"` |
| `super().save(...)` | Calls Django's original save, persisting data to the DB |
| `*args, **kwargs` | Forwards any extra arguments to the parent method |

### What is `slugify`?

`slugify` is a built-in Django utility from `django.utils.text`. It transforms a plain string into a valid slug:

```python
from django.utils.text import slugify

slugify("Harry Potter One")
# → "harry-potter-one"
```

## Step 3: Create and Run Migrations

Any time you change a model (add a field, modify a field, etc.), you must create and apply migrations to keep the database schema in sync.

<Steps>

    ### Make migrations

    ```bash
    python manage.py makemigrations
    ```

    ### Apply migrations

    ```bash
    python manage.py migrate
    ```

</Steps>

<Callout type="error">
    Skipping migrations after a model change will cause your model and database to fall out of sync, which leads to errors at runtime.
</Callout>

---

## Step 4: Update Existing Records

The overridden `save()` only runs on new saves going forward. Existing database records will not have their slug auto-set until you re-save them.

Use the Django shell to update existing entries manually:

```bash
python manage.py shell
```

```python
from django.utils.text import slugify

for book in Book.objects.all():
    book.slug = slugify(book.title)
    book.save()

# Verify the slug was set
print(book.slug)  # → "harry-potter-one"
```

<Callout type="info">
    In a real project, you'd define your model fully before seeding data, so this manual step wouldn't be needed.
</Callout>

## Full Model Example

```python filename="book_outlet/models.py"
from django.db import models
from django.utils.text import slugify


class Book(models.Model):
    title = models.CharField(max_length=200)
    rating = models.IntegerField()
    author = models.CharField(max_length=100, null=True)
    is_bestselling = models.BooleanField(default=False)
    slug = models.SlugField(unique=True)

    def save(self, *args, **kwargs):
        self.slug = slugify(self.title)
        super().save(*args, **kwargs)

    def __str__(self):
        return f"{self.title} ({self.rating})"
```
