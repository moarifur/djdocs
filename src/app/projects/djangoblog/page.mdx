# Build a Complete Django Blogging System

A comprehensive guide to building a full-featured, multi-role Django blog application — covering models, authentication, permissions, dashboards, and deployment.

<Callout type="info">
    This guide is based on the **Tech with Ratan** premium Django project course. By the end, you'll have a portfolio-ready blogging system with role-based access control, a custom dashboard, and a live deployment on Python Anywhere.

## What You'll Build

The final project includes:

- A dynamic homepage with featured posts, recent articles, and category filtering
- SEO-friendly **slug-based URLs** (no raw primary keys)
- A **multi-role system**: Super Admin → Manager → Editor
- A **custom dashboard** where managers and editors manage posts, categories, and users
- A **comment system** restricted to authenticated users
- Full **CRUD operations** for posts, categories, and users
- **Deployment** to Python Anywhere (free tier)

## Foreign Key Relationships

Understanding the **many-to-one** (ForeignKey) relationships in this project:

| Relationship | Rule |
|---|---|
| Category → Blog | One category can have **many** blog posts |
| Blog → Category | Each blog post belongs to **one** category |
| User → Blog | One user (author) can write **many** posts |
| Blog → User | Each post is written by **one** author |
| User → Comment | A user can write **many** comments |
| Blog → Comment | A blog can have **many** comments |

<Callout type="info">
    When you use a ForeignKey field (e.g. `category`), Django stores the **ID** of the related object — not its name. In the database table the column will be named `category_id`.
</Callout>

## Displaying Posts on the Homepage

In `blog_main/views.py`:

```python
from django.shortcuts import render
from blocks.models import Blog, Category

def home(request):
featured_posts = Blog.objects.filter(is_featured=True, status='published').order_by('-updated_at')
posts          = Blog.objects.filter(is_featured=False, status='published')

context = {
'featured_posts': featured_posts,
    'posts':          posts,
}
return render(request, 'home.html', context)
```

In `home.html` — display the first featured post as the cover image:

```django
{% for post in featured_posts %}
{% if forloop.first %}
<div style="background-image: url('{{ post.featured_image.url }}')">
    <h2>{{ post.title }}</h2>
    <p>{{ post.short_description|truncatewords:25 }}</p>
</div>
{% endif %}
{% endfor %}
```

Loop through remaining featured posts:

```django
{% for post in featured_posts %}
{% if not forloop.first %}
<h5>{{ post.title }}</h5>
<small>{{ post.created_at|timesince }} ago</small>
{% endif %}
{% endfor %}
```

<Callout>
    Use `|truncatewords:25` to limit description length, and `|timesince` to display human-readable timestamps like *"2 hours ago"*.
</Callout>

## Template Inheritance

Instead of copy-pasting headers and footers across every template, use a single `base.html`.

**`templates/base.html`** — holds the shared layout:

```django
<!DOCTYPE html>
<html>
<head>...</head>
<body>
<!-- Header / Navbar -->
<nav>...</nav>

<!-- Dynamic content goes here -->
{% block content %}{% endblock %}

<!-- Footer -->
<footer>...</footer>
</body>
</html>
```

**Every other template** just extends it:

```django
{% extends 'base.html' %}

{% block content %}
<!-- Page-specific content -->
{% endblock %}
```

<Callout type="tip">
    If you need to update the navbar, change it **once** in `base.html` and every page reflects it instantly — no more hunting through 10 files.
</Callout>

## Context Processors

To make `categories` and `social_links` available on **every** page without repeating queries in each view, use a context processor.

Create `blocks/context_processors.py`:

```python
from .models import Category
from assignments.models import SocialLink

def get_categories(request):
categories = Category.objects.all()
return {'categories': categories}

def get_social_links(request):
social_links = SocialLink.objects.all()
return {'social_links': social_links}
```

Register both in `settings.py`:

```python
TEMPLATES = [{
'OPTIONS': {
    'context_processors': [
        ...
            'blocks.context_processors.get_categories',
        'blocks.context_processors.get_social_links',
    ],
},
}]
```

    ```python
    ]
    ```

    ### Write the view

    ```python
    from django.shortcuts import render, get_object_or_404, redirect
    from .models import Blog, Category

    def post_by_category(request, category_id):
    category = get_object_or_404(Category, pk=category_id)
    posts    = Blog.objects.filter(status='published', category=category_id)
    context  = {'posts': posts, 'category': category}
    return render(request, 'post_by_category.html', context)
    ```

## Single Blog Post Page

<Steps>

    ### Add URL pattern

    In `blog_main/urls.py`:

    ```python
    path('blocks/<slug:slug>/', blocks_view.blocks, name='blocks'),
    ```

    ### Write the view

    ```python
    from django.shortcuts import get_object_or_404
    from .models import Blog, Comment

    def blocks(request, slug):
    single_blog   = get_object_or_404(Blog, slug=slug, status='published')
    comments      = Comment.objects.filter(blog=single_blog)
    comment_count = comments.count()

    if request.method == 'POST':
    comment        = Comment()
    comment.user   = request.user
    comment.blog   = single_blog
    comment.comment = request.POST.get('comment')
    comment.save()
    from django.http import HttpResponseRedirect
    return HttpResponseRedirect(request.path_info)

    context = {
        'single_blog':   single_blog,
        'comments':      comments,
        'comment_count': comment_count,
    }
    return render(request, 'blocks.html', context)
    ```

    ### Link post titles on the homepage

    ```django
    <a href="{% url 'blocks' post.slug %}" class="text-dark">{{ post.title }}</a>
    ```

</Steps>

## Search Functionality

<Steps>

    ### Add the URL

    ```python
    path('blocks/search/', blocks_view.search, name='search'),
    ```

    ### Write the view

    ```python
    from django.db.models import Q

    def search(request):
    keyword = request.GET.get('keyword')
    blocks  = Blog.objects.filter(
    Q(title__icontains=keyword) |
    Q(short_description__icontains=keyword) |
    Q(blog_body__icontains=keyword),
    status='published'
    )
    context = {'blocks': blocks, 'keyword': keyword}
    return render(request, 'search.html', context)
    ```

    ### Add the search form to `base.html`

    ```django
    <form action="{% url 'search' %}" method="get">
        <div class="input-group">
            <input type="text" name="keyword" value="{{ keyword }}" placeholder="Search...">
                <button type="submit" class="btn btn-warning">Go</button>
        </div>
    </form>
    ```

</Steps>

<Callout>
    The `Q` object enables **OR** lookups. A comma (`,`) between conditions acts as **AND**.
</Callout>

### Login & Logout

```python
from django.contrib.auth.forms import AuthenticationForm
from django.contrib import auth

def login(request):
if request.method == 'POST':
form = AuthenticationForm(request, data=request.POST)
if form.is_valid():
username = form.cleaned_data.get('username')
password = form.cleaned_data.get('password')
user     = auth.authenticate(username=username, password=password)
if user is not None:
auth.login(request, user)
return redirect('dashboard')
else:
form = AuthenticationForm()
return render(request, 'login.html', {'form': form})

def log_out(request):
auth.logout(request)
return redirect('home')
```

## Permissions & Groups

Django auto-generates four permissions for every model: `add`, `change`, `delete`, `view`.

### Roles in this project

| Role | Permissions |
|---|---|
| **Super Admin** | Everything |
| **Manager** | Manage users + all blog/category permissions |
| **Editor** | Blog post & category CRUD only |

### Creating a Manager group

In the Django admin panel:

1. Go to **Groups** → **Add Group**
2. Name it `manager`
3. Select permissions: `auth | user | Can add/change/delete/view user` + blog/category permissions
4. Save and assign users to this group

### Checking permissions in templates

```django
{# Show "Users" sidebar link only to managers #}
{% if perms.auth.view_user %}
<a href="{% url 'users' %}">Users</a>
{% endif %}
```

## Blog Post CRUD

### The Post Form

```python
# dashboards/forms.py
from django import forms
from blocks.models import Blog

class BlogPostForm(forms.ModelForm):
class Meta:
model  = Blog
# Exclude author and slug — these are set programmatically
fields = ['title', 'category', 'featured_image',
'short_description', 'blog_body', 'status', 'is_featured']
```

<Callout>
    **Never expose `author` or `slug` fields to editors.** The author should always be the logged-in user, and the slug is generated from the title automatically.
</Callout>

### Adding a Post with Auto-Slug

```python
from django.template.defaultfilters import slugify

def add_post(request):
if request.method == 'POST':
form = BlogPostForm(request.POST, request.FILES)  # FILES for images!
if form.is_valid():
post        = form.save(commit=False)   # don't save yet
post.author = request.user              # assign logged-in user
post.save()                             # save to get an ID
title      = form.cleaned_data['title']
post.slug  = slugify(title) + '-' + str(post.id)  # unique slug
post.save()
return redirect('post')
else:
form = BlogPostForm()
return render(request, 'dashboard/add_post.html', {'form': form})
```

<Callout type="warning">
    Always add `enctype="multipart/form-data"` to any `<form>` that contains an image or file field, and pass `request.FILES` alongside `request.POST` in the view.
</Callout>

### Editing a Post

```python
def edit_post(request, pk):
post = get_object_or_404(Blog, pk=pk)
```

        ```python
        def add_user(request):
        if request.method == 'POST':
        form = AddUserForm(request.POST)
        if form.is_valid():
        form.save()
        return redirect('users')
        else:
        form = AddUserForm()
        return render(request, 'dashboard/add_user.html', {'form': form})

        def edit_user(request, pk):
        user = get_object_or_404(User, pk=pk)
        form = EditUserForm(request.POST or None, instance=user)
        if form.is_valid():
        form.save()
        return redirect('users')
        return render(request, 'dashboard/edit_user.html', {'form': form})

        def delete_user(request, pk):
        user = get_object_or_404(User, pk=pk)
        user.delete()
        return redirect('users')
        ```

## Deployment on Python Anywhere

<Callout type="info">
    Python Anywhere offers a **free tier** that hosts one Django application — perfect for portfolio projects.
</Callout>

<Steps>

    ### Prepare your codebase

    Remove `db.sqlite3` and `media/` from `.gitignore`, update `requirements.txt`, then push:

    ```bash
    pip freeze > requirements.txt
    # Edit requirements.txt — add version marker for backports.zoneinfo if on Python < 3.9:
    # backports.zoneinfo>=0.2.1; python_version < "3.9"

    git add -A
    git commit -m "Deployment"
    git push origin main
    ```

    ### Clone the repo on Python Anywhere

    Open a **Bash console** on Python Anywhere:

    ```bash
    git clone https://github.com/YOUR_USERNAME/django-blog.git
    cd django-blog
    ```

    ### Create a virtual environment and install packages

    ```bash
    mkvirtualenv my-site-virtualenv --python=python3.10
    pip install -r requirements.txt
    ```

    ### Configure the Web App

    In the Python Anywhere **Web** tab:
    - Click **Add a new web app** → **Manual configuration** → **Python 3.10**
    - Set **Source code**: `/home/YOUR_USERNAME/django-blog`
    - Set **Virtualenv**: `/home/YOUR_USERNAME/.virtualenvs/my-site-virtualenv`
    - Set **Static files**: URL `/static/` → Directory `/home/YOUR_USERNAME/django-blog/static`

    ### Edit the WSGI file

    Paste this into the auto-generated WSGI file (replacing the existing content):

    ```python
    import sys, os
    path = '/home/YOUR_USERNAME/django-blog'
    if path not in sys.path:
    sys.path.insert(0, path)
    os.environ['DJANGO_SETTINGS_MODULE'] = 'blog_main.settings'
    from django.core.wsgi import get_wsgi_application
    application = get_wsgi_application()
    ```

    ### Collect static files and finalize settings

    Back in the Bash console:

    ```bash
    python manage.py collectstatic
    ```

    In `blog_main/settings.py` (edit via the Files tab):

    ```python
    DEBUG        = False
    ALLOWED_HOSTS = ['YOUR_USERNAME.pythonanywhere.com']

    # Fix for Python Anywhere template resolution
    TEMPLATES = [{
    'DIRS': ['/home/YOUR_USERNAME/django-blog/templates'],
    ...
}]
    ```

    ### Reload the app

    Click **Reload** in the Web tab. Your site is live at `https://YOUR_USERNAME.pythonanywhere.com` 🎉

</Steps>

<Callout type="tip">
    If you see a **500 error**, temporarily set `DEBUG = True` to read the full traceback in the browser, fix the issue, then set it back to `False`.
</Callout>

## Key Takeaways

| Concept | What You Learned |
|---|---|
| **Models & ForeignKey** | Relational data: blog → category, blog → author, comment → blog |
| **Slug** | SEO-friendly URLs generated with `slugify` + post ID for uniqueness |
| **Context Processors** | Share data (categories, social links) across all templates without repeating queries |
| **Template Inheritance** | `base.html` + `{% block content %}` eliminates copy-paste |
| **Model Forms** | Quickly build forms from model fields; use `commit=False` to add custom data before saving |
| **File Uploads** | `enctype="multipart/form-data"` on the form + `request.FILES` in the view |
| **Authentication** | Django's built-in `auth.login`, `auth.logout`, `auth.authenticate` |
| **Permissions & Groups** | Fine-grained access control with `perms.app.permission` template tags |
| **login_required** | Decorator that redirects unauthenticated users away from protected views |
| **Q Objects** | OR-based database lookups for full-text search across multiple fields |
| **Deployment** | Python Anywhere: clone → virtualenv → collectstatic → WSGI → reload |
