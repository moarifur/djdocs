# Getting Started with Django

Before you write a single line of Django code, two things need to be in place — a clean Python environment and a working Django project.

A **virtual environment** keeps your project's dependencies isolated, so installing Django here doesn't affect anything else on your machine. Once that's set up, Django gives you a `startproject` command that scaffolds everything you need to begin.

From there, Django organises your code into **apps** — small, self-contained modules that each own a specific slice of your project. An online shop might have a `products` app, a `cart` app, and an `admin` app. Each one is independent, reusable, and easy to reason about.

---

## Learning Objectives

By the end of this section you should be able to:

- Explain what a virtual environment is and why it matters
- Set up Django from scratch — Python, venv, install, and first project
- Navigate the files Django generates and know what each one does
- Create a new app and register it in `settings.py`
